# Day 9: NutriScope Comparative Suite - Evolution of a Feature-Rich Analytics Dashboard

## 📊 Objective
For Day 9, I engineered and evaluated two distinct phases of **NutriScope**, translating nutritional data models into actionable biometric analytics:
1. **MVP Engine (Prompt 1):** Focus on core input logic, BMR-based dynamic nutrient targeting, and localized dietary data handling (20 items).
2. **Enhanced Pro Framework (Prompt 2):** Integrating complex medical-grade risk engines, full-scale asset expansion (60+ items), CSV pipeline automated ingestion, and adaptive meal planners.

To facilitate assessment, I developed a dual-view application that presents these frameworks side-by-side on a single responsive pane, including an "Optimize for Screenshot" controller to capture the entire dashboard state without scrolling.

---

## 💻 Tech Stack & Deployment
*   **Structure & Styling:** HTML5, Tailwind CSS
*   **Analytics Engine:** JavaScript (ES6+), Chart.js
*   **Asset References:** ICMR-NIN (Indian Council of Medical Research - National Institute of Nutrition, 2024), USDA FoodData Central.
*   **Version Control:** Git & GitHub

---

## 📈 Functional Comparison Matrix

| Core Feature Area | Phase 1: MVP (Prompt 1) | Phase 2: Enhanced Pro (Prompt 2) |
| :--- | :--- | :--- |
| **Nutrient Assets** | Standard 20 Items | Expanded 60+ Assets (incl. high-protein alternatives, healthy fats). |
| **BMR Engine** | Static input logic for simple targets. | Dynamic multi-variable targets based on metabolic profiles. |
| **Physiological Risks** | Macro balance percentage charts. | Multi-system Risk Analyzer (Renal Stress & Gut Health Index). |
| **Data Ingestion** | Manual per-item logging. | Mock CSV/Log parsing automated pipeline. |
| **Meal Optimization** | Absent. | Adaptive 2-Phase Clinical Meal Planner with strict constraints. |
| **UI Aesthetics** | Clean Dark Minimalist SaaS Interface. | High-Fidelity Medical Diagnostic Dashboard Layout. |

---

## 🎓 Key Engineering Learnings
*   **Algorithmic Biometrics:** Successfully modeled Renal Load and Gut Health risks based on complex nutrient intake ratios from varied dietary sources.
*   **State Management Scaling:** Managed dynamic application state transitions across two highly detailed, isolated dashboard components within a single view.
*   **Responsive UI/UX:** Learned to optimize premium dark-themed interfaces for laptop screenshots, preventing information loss from excessive scrolling.
*   **Automated Data Flow:** Designed a mock framework for CSV ingestion that automatically populates analytics engines, reducing user friction in data logging.
